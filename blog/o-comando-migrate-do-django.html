O comando migrate

Quando executamos o comando makemigrations, o Django cria o banco de dados e as migrations, mas no as executa, isto : no realmente aplica as alteraes no banco de dados.

Para que o Django as aplique, so necessrias trs coisas, basicamente:

  • 1. Que a configurao da interface com o banco de dados esteja descrita no arquivo de configuraes settings.py.
  • 2. Que os modelos e migrations estejam definidos para esse projeto.
  • 3. Execuo do comando migrate.

Se voc acompanhou os ltimos posts, voc criou o projeto Hello World, Django atravs do comando django-admin.py createproject helloworld e, portanto, a configurao padro foi aplicada.

Opes do comando migrate

O formato padro do comando migrate o seguinte:

Abaixo vamos ver as opes do comando.

  • Sem argumentos: aplica todas as migraes disponveis de todos os apps definidos em INSTALLED_APPS.
  • <label_app>: Aplica-se todas as migraes do app especificado at a mais recente. Pode envolver a aplicao de outras migraes, pois pode haver dependncia entre migraes.
  • <label_app> <nome_migrations>: Leva a estrutuda do banco de dados um estado em que a migrao aplicada, sem aplicar as migraes seguintes.
  • --database DATABASE: Especifica o banco de dados para executar as migraes. As opes disponves so quelas que foram definidas pela configurao DATABASES no settings.py.
  • --fake: Opo avanada que permite marcar migraes como aplicadas, sem realmente aplic-las (ou seja, sem modificar a estrutuda do seu banco de dados). Opo geralmente utilizada para recuperao da base de dados, ou quando se quer alterar manualmente a tabela interna de estados das migraes do Django.
  • --fake-initial: Permite ao Django pular a migrao inicial, caso todas as tabelas j tenham sido previamente criadas. Essa opo usada quando a estrutura inicial de tabelas j foi criada previamente no banco de dados (por exemplo, sistema legado ou se um DBA j executou um script de criao das tabelas).
  • --plan: Apenas mostra as migraes que sero aplicadas com a execuo do comando migrate.
  • --run-syncdb: Permite a criao da estrutura de tabelas sem as migraes. Essa opo no verifica a estrutura das tabelas nem verifica a questo das dependncias entre elas. Essa opo usada com sistemas muito grandes, com centenas de modelos, onde o framework de migraes do Django pode ser lento.
  • --noinput ou --no-input: Evita confirmaes de usurio. Opo til quando essa opo for utilizada em scripts bash.
